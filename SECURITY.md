# Security Policy

## Supported versions

Security fixes are provided for the latest state of the `main` branch.

## Reporting a vulnerability

1. Do not open a public issue with exploit details.
2. Open a private GitHub security advisory for this repository.
3. If private advisories are unavailable, open a regular issue with minimal details and request a private contact channel.

Please include:

- affected endpoint or feature
- reproduction steps
- expected impact
- proof-of-concept (if available)

I will acknowledge new reports as soon as possible and prioritize critical vulnerabilities first.
